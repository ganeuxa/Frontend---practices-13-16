# 📝 Заметки (Notes PWA)

## 📖 Описание проекта

Progressive Web Application (PWA) для управления заметками с поддержкой офлайн-режима, push-уведомлений, WebSocket синхронизации в реальном времени и расширенного функционала напоминаний (планирование, откладывание).

Проект реализован в рамках практических занятий №13–17 по дисциплине «Фронтенд и бэкенд разработка».

---

## 🛠 Технологии

| Компонент | Технологии |
|-----------|------------|
| **Frontend** | HTML5, CSS3 (Chota), Vanilla JavaScript |
| **Backend** | Node.js, Express, Socket.IO |
| **PWA** | Service Worker, Cache API, Web App Manifest |
| **Notifications** | Push API, Web Push Protocol |
| **Security** | HTTPS (self-signed certificates) |
| **Storage** | LocalStorage (client), Map (server memory) |

---

## 📚 Реализованные практики

### ✅ Практика 13: Service Worker
- Реализована стратегия кэширования **App Shell**.
- Обеспечена работа приложения в офлайн-режиме.
- Настроена пре-кэширование статических ресурсов (HTML, CSS, JS, иконки).

### ✅ Практика 14: Web App Manifest
- Добавлен файл `manifest.json` для установки приложения на домашний экран.
- Настроены иконки, тема оформления и режим отображения (`standalone`).

### ✅ Практика 15: HTTPS + App Shell
- Развёрнуто безопасное соединение через **HTTPS** (localhost сертификаты).
- Реализована архитектура **App Shell**: мгновенная загрузка интерфейса из кэша с последующей подгрузкой контента.

### ✅ Практика 16: WebSocket + Push Notifications
- Настроено двустороннее соединение через **Socket.IO** для синхронизации заметок между клиентами в реальном времени.
- Реализована подписка на push-уведомления через **Push Manager**.
- На сервере реализована отправка уведомлений через библиотеку `web-push`.

### ✅ Практика 17: Детализация Push (Напоминания)
- **Планирование уведомлений:** Добавлена возможность создания заметок с указанием даты и времени напоминания.
- **Серверный планировщик:** На бэкенде реализовано хранение активных таймеров напоминаний (`Map` + `setTimeout`).
- **Действия в уведомлении:** В push-уведомление добавлена кнопка **«⏰ Отложить на 5 минут»**.
- **Обработка в Service Worker:** Реализован обработчик `notificationclick` для действия `snooze`, отправляющий запрос на сервер для переноса времени уведомления.
- **Синхронизация:** Статус напоминания («Отложено») синхронизируется между сервером и клиентом в реальном времени через WebSocket события.
- **Работа в фоне:** Уведомления приходят даже при закрытой вкладке браузера.

## 🚀 Установка и запуск

### 1. Требования
- Node.js (версия 18+)
- Браузер с поддержкой PWA и Push API (Chrome, Opera, Edge)

### 2. Установка зависимостей
```bash
# В корневой папке проекта
npm install

# В папке server
cd server
npm install
```

### 3. Настройка HTTPS
Для локальной разработки используются самоподписанные сертификаты. Убедитесь, что в папке `server/` присутствуют файлы:
- `localhost+2-key.pem`
- `localhost+2.pem`

*(Если сертификаты отсутствуют, сгенерируйте их через OpenSSL или аналогичный инструмент)*

### 4. Запуск серверов
Проект использует раздельную архитектуру фронтенда и бэкенда. Необходимо запустить два процесса:

**Терминал 1 (Бэкенд + WebSocket + Push):**
```bash
cd server
node server_push.js
```
*Сервер запустится на:* `https://localhost:3001`

**Терминал 2 (Фронтенд):**
```bash
cd server
node server_frontend.js
```
*Сервер запустится на:* `https://localhost:4001`

### 5. Открытие приложения
Перейдите в браузере по адресу: **`https://localhost:4001`**

> ⚠️ **Важно:** При первом открытии браузер может предупредить о небезопасном соединении (из-за самоподписанных сертификатов). Необходимо принять риск и продолжить. Для корректной работы Service Worker и Push обязательно используйте **HTTPS**.

## 🧪 Как тестировать функционал

### 1. Создание заметки
1. Введите текст в поле «Введите текст заметки».
2. Нажмите **«Добавить»**.
3. Заметка появится в списке и сохранится в `localStorage`.

### 2. Создание напоминания
1. Заполните форму **«Текст напоминания»** и выберите дату/время в будущем.
2. Нажмите **«С напоминанием»**.
3. В списке заметок появится метка времени напоминания.

### 3. Проверка Push-уведомлений
1. Нажмите кнопку **«Включить уведомления»** внизу страницы и разрешите их в браузере.
2. Создайте заметку с напоминанием на 1–2 минуты вперёд.
3. **Закройте вкладку** или сверните браузер.
4. Дождитесь системного уведомления.

### 4. Проверка функции «Отложить» (Snooze)
1. Когда придёт уведомление, нажмите на кнопку **«⏰ Отложить на 5 мин»** (не кликайте по тексту уведомления).
2. Появится подтверждение об откладывании.
3. В списке заметок статус изменится на **«⏰ Отложено на 5 минуточек»**.
4. Через 5 минут придёт повторное уведомление.

### 5. Офлайн-режим
1. Откройте приложение, дождитесь загрузки.
2. Отключите интернет.
3. Обновите страницу — приложение загрузится из кэша Service Worker.

## 🔑 VAPID Ключи

В проекте используются публичные/приватные ключи для Web Push.
- **Public Key:** Встроен в `app.js` и `server_push.js`.
- **Private Key:** Используется только на сервере для подписи уведомлений.

*(Для продакшена рекомендуется сгенерировать новые ключи через `web-push generate-vapid-keys`)*
